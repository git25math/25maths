# Communication Transfer

## 用户核心意图（已落地）
1. 学生体验优先：学生页必须有足够答题空间。
2. 版面约束明确：理想 3 页（1-2 学生页、3 答案页）。
3. 执行策略固定：遇到问题必须即时修复，不盲目追量。
4. 质量底线：严格按考纲，题目相关，答案正确。
5. 管理诉求：把任务独立出来，避免与并行任务互相干扰。

## 对话中形成的“硬规则”
- Rule-1：Fail Fast，Stop on First Failure。
- Rule-2：修复后必须更新生成逻辑与验收逻辑。
- Rule-3：修复后必须复测，且可追踪。
- Rule-4：full-pack 标准固定：student PDF 2 页、pack PDF 3 页。
- Rule-5：跨考纲污染（board/tier/domain）必须归零。

## 已转移的指挥语义
- “总指挥官”职责：任务拆解、执行编排、质量把关、失败闭环、进度报告。
- “多 agent”职责：生成/修复、题目质量校验、PDF 与版式校验并行协作。
