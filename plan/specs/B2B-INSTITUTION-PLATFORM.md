# B2B Institution Platform

> Status: Revised for the post-exercise website.
> Created: 2026-02-27.
> Revised: 2026-05-03.

The original B2B specification included an exercise-catalog homework flow that depended on the retired website exercise product line. That flow has been removed from the active site.

## Current Institution Scope

The institution surface should focus on resource planning:

- Institution profile and member management.
- Class/student roster records where already supported.
- Worksheet/Kahoot/resource-pack planning.
- Follow-up status, due dates, and teacher notes.
- Links to active resource pages and Payhip/member downloads.

## Retired Scope

- No exercise-catalog homework creation.
- No exercise-player submission tracking.
- No dependency on retired website exercise routes, layouts, runtime JavaScript, Functions API routes, or telemetry tables.

## Future Direction

If a new question-practice platform is introduced later, institution analytics should integrate through a deliberate API or aggregate table. The website institution pages should not directly own raw question attempts or resurrect the deleted exercise catalog.
